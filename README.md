## 방식

### 스터디 이전

1. 정해진 분량의 책 공부하고, `내용 정리+질문 사항`을 하나의 md 파일로 정리
2. 깃허브 레파지토리에 branch 생성하고, md 파일 업로드
3. 해당 branch 업로드 내용에 대한 Pull Request 전송
4. 스터디장이 Issue에 모든 스터디원의 질문 사항 모아서 업로드
5. 스터디원들은 질문 사항에 대한 답변 준비

### 스터디 당일

1. 발표자가 해당 주차 내용 발표
2. 발표 후, 사전 질문에 대한 질의응답
